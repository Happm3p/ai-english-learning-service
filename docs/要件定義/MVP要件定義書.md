# AI英語学習サービス MVP要件定義書

## 1. サービス概要
- **サービス名**: (仮) AI English Corrector
- **コンセプト**: ユーザーが入力した英文をAI (Claude Haiku 4.5) が即座に添削・フィードバックし、自然で正確な英語表現を習得できるサービス。
- **ターゲット**: 英語学習中の社会人・学生
- **技術スタック**: Next.js 15, App Router, MUI, MongoDB, Mongoose, NextAuth.js, Nodemailer

## 4. ユーザーニーズ洗い出し

### 機能的ニーズ
- **即時性**: 入力したら待たずにすぐに結果を知りたい（Claude Haiku 4.5の高速性を活用）。
- **正確性**: 文法ミスだけでなく、文脈に合った自然な表現を教えてほしい。
- **解説**: なぜ間違っているのか、なぜその修正案が良いのかの理由を知りたい。
- **保存**: 自分が書いた文章と添削結果を後で見返したい。
- **手軽さ**: スマホでもPCでも使いやすいUIが良い。

### 情緒的ニーズ
- **安心感**: 自分の英語が恥ずかしくないレベルだと確認したい。
- **成長実感**: 以前よりミスが減った、表現の幅が広がったと感じたい。
- **継続性**: 孤独な学習でもモチベーションを保ちたい。

## 5. MVP機能一覧と優先順位

### 優先度定義
- **P0 (Must)**: MVPとして必須。これがないとサービスとして成立しない。
- **P1 (Should)**: リリース時にあることが望ましい。
- **P2 (Nice to have)**: 将来的に実装したい。

| カテゴリ | 機能名 | 概要 | 優先度 | 実装技術メモ |
| :--- | :--- | :--- | :--- | :--- |
| **認証** | ユーザー登録・ログイン | メールアドレス/パスワード、Google認証 | **P0** | NextAuth.js |
| **添削** | 英文入力フォーム | ユーザーが英文を入力するエリア。文字数カウントなど。 | **P0** | MUI (TextField) |
| **添削** | AI添削実行 | Claude Haiku 4.5 APIを呼び出し、修正案と解説を生成。 | **P0** | Server Actions / API Route |
| **添削** | フィードバック表示 | 修正前後の比較表示、解説、代替表現の提示。 | **P0** | MUI (Card, Typography) |
| **履歴** | 学習履歴一覧 | 過去の添削内容をリスト表示。 | **P1** | MongoDB, Mongoose |
| **履歴** | 履歴詳細閲覧 | 過去の特定の添削結果を再確認できる。 | **P1** | Dynamic Routing |
| **通知** | 学習リマインダー | 一定期間利用がない場合などにメールを送信。 | **P2** | Nodemailer |
| **設定** | プロフィール設定 | ユーザー名、学習目標の設定（添削のトーン調整に使用）。 | **P2** | MongoDB |
| **添削** | 添削モード選択 | 「ビジネス」「カジュアル」「アカデミック」などトーン指定。 | **P2** | Prompt Engineering |

### MVPスコープ（P0 + P1の一部）
1.  **認証機能**: ユーザーを識別し、個人の履歴を保存するために必須。
2.  **メイン添削機能**: 英文入力 -> Claude Haiku 4.5による高速添削 -> 結果表示（修正文 + 解説）。
3.  **履歴機能（簡易）**: 自分の過去の添削ログが見られること。復習に必須。

※ メール通知（Nodemailer）は技術スタックに含まれているが、MVPのコア体験（添削）には必須ではないため、P2またはP1の後半とする。ただし、登録時の確認メール等で早期に使用する可能性はある。
